# 90 天内容与质量计划

周期：2026-07-17 至 2026-10-15

## 总原则

- 前 8 周优先修好现有页面，不用新增数量制造“更新感”。
- 新页面必须由真实用户任务、Search Console 查询或明确内容缺口触发。
- 每周最多发布 1 至 2 个新页面，同时更新 3 至 5 个现有页面。
- 新主题若能通过扩展现有页面解决，优先更新或合并，不拆成重复文章。
- 每个发布任务同时指定官方来源、复核周期和停止发布条件。

## 候选内容的发布条件

以下条件至少满足两项，才考虑新建页面：

1. Search Console 在 28 天内出现稳定、同意图查询，并且现有页面无法完整回答。
2. 读者纠错或实际办事反馈显示现有路径存在明显缺口。
3. 官方机构推出新流程、证件类型、期限或入口。
4. 一个高风险问题需要单独的来源映射和免责声明，放在综合页会降低清晰度。

不满足条件时，更新现有页面或记录为观察项。
