# 月度 Search Console 内容决策流程

首次执行时间：站点积累至少 28 天有效数据后。

## 需要的数据

- Search results：最近 28 天与前 28 天对比。
- 维度：Query、Page、Country、Device。
- Indexing：未索引、抓取异常、重复 canonical 和软 404。
- Core Web Vitals：移动端优先。

不需要用户提供 Google 密码。执行时通过用户已经登录的浏览器查看，或由用户导出 CSV。

## 每月步骤

1. 导出最近 28 天查询和页面数据。
2. 删除品牌词和明显无关查询，保留真实办事意图。
3. 按页面合并查询，识别同一页面承担的主要和次要任务。
4. 对比展示、点击、CTR 和平均排名的变化。
5. 把页面放入以下行动队列。

## 行动队列

| 队列 | 信号 | 行动 |
| --- | --- | --- |
| 保持 | 查询与页面意图一致，点击和排名稳定 | 只做来源与链接维护 |
| 补答案 | 有展示，用户查询的问题正文没有直接回答 | 增加清晰章节、FAQ 或来源映射 |
| 改标题 | 排名尚可但 CTR 明显偏低，标题与查询不完全匹配 | 重写 title / description，不夸大 |
| 合并 | 多页争夺同一查询，内容重复 | 合并到更完整页面并重定向 |
| 新建 | 查询稳定、任务独立、现有页无法承载 | 完成选题门禁后新建 |
| 刷新 | 展示或排名下跌，同时官方规则或来源发生变化 | 重新核验事实并记录变更 |
| 停止索引 | 页面无独立任务、长期无价值或只有重复目录 | 合并、noindex 或删除 |

## 决策阈值

阈值用于排序，不是机械发布规则：

- 展示不少于 50 且排名 5 至 20：优先补答案和内部链接。
- 排名 1 至 10 但 CTR 显著低于同类页面：检查标题是否准确清楚。
- 两个页面对同组查询都有展示：检查意图冲突和内容重复。
- 连续两个 28 天周期无展示且无直接用户价值：进入合并或停止索引评估。
- 规则变化、高风险错误或失效入口不受流量阈值限制，立即处理。

## 不能做的事

- 不因为某个词有展示就批量生成近义词页面。
- 不为了“新鲜度”修改日期。
- 不删除没有流量但承担重要安全、政策或导航功能的页面。
- 不把平均排名小幅波动当成必须重写全部内容的信号。
- 不通过虚假标题、保证性措辞或隐藏风险提高点击率。
